## 一、扩展面板：Chrome 小恐龙跳跃游戏

### 背景
用户询问在「扩展面板」中实现类 Chrome 恐龙跳跃游戏的可行性。评估结论：约 250 行，难度低，随即实施。

### 技术架构

| 项目 | 实现方式 |
|------|----------|
| 渲染 | `<canvas id="dino-canvas">` + Canvas 2D API |
| 封装 | IIFE（`(function(){ 'use strict'; ... })()`），避免全局污染 |
| 物理 | 重力 `GR=0.55`；跳跃初速动态计算：`jV = -√(2·GR·availableSpace)` |
| 碰撞 | AABB 矩形碰撞检测，hitbox 由 `CHW`（半宽）× `CHH`（高）定义 |
| 响应式 | `ResizeObserver` 在 idle 状态监听面板尺寸变化，动态重设 canvas |
| 障碍物 | 暗色晶体/源石地形，蓝色辉光，多边形路径绘制 |
| 输入 | 键盘 Space/ArrowUp；canvas 单击；触屏 touchstart |
| 游戏状态 | `idle` → `running` → `dead` → `idle`（点击/按键重置） |
| 分数 | 帧数累计，HI 分记忆本次会话最高分 |

### 主要常量（最终版）

```javascript
const GH  = 22;   // 地面高度
const CVH = 60;   // 角色脚到画布底部的偏移（用于跳跃计算）
const CHH = 52;   // 角色碰撞箱高度
const CHW = 11;   // 角色碰撞箱半宽
const CX  = 76;   // 角色水平位置（固定）
const GR  = 0.55; // 重力加速度
```

## 二、角色迭代

### 第一版：自定义动漫角色（已弃用）

用户提供参考图（明日方舟角色，双马尾、蓝色犄角、灰色夹克"5"、白裙、龙尾，朝右），以几何图形绘制。
常量：`CVH=97, CHH=66, CHW=13`

实现特征：双马尾（梯形）、蓝色犄角、红色发夹、灰色夹克（带"5"文字）、白裙摆、黑靴、橙色龙尾。

### 第二版：Chrome T-Rex（当前版本）

用户要求替换为谷歌浏览器标准小恐龙，保留全部游戏逻辑不变。

**变更内容**：
1. 常量调整：`CVH 97→60`，`CHH 66→52`，`CHW 13→11`
2. `drawChar()` 函数全量替换

**T-Rex 绘制规格**（颜色 `#535353`，脚底为 y=0，朝右）：

| 部件 | 绘制方式 |
|------|---------|
| 尾巴 | 梯形多边形，向左延伸 |
| 后腿 | 矩形 + 脚掌矩形，跑步时交替摆动 |
| 躯干 | 主矩形 |
| 前臂 | 小矩形 stub |
| 头颅脊背 | 较高矩形（头顶突起） |
| 头部主体 | 矩形 |
| 下颌/吻部 | 矩形 |
| 前腿 | 矩形 + 脚掌矩形，跑步时与后腿反相摆动 |
| 嘴缝动画 | 跑步时 `sin` 驱动开合，跳跃时闭嘴，`fillStyle='#1a1a1a'` |
| 眼睛（存活） | 白色底色 + 黑色瞳孔 |
| 眼睛（死亡） | X 形，`strokeStyle='#f5f5f5'` |
