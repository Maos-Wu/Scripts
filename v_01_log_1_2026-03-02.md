# 开发日志 v0.1 — 中国股市实时终端监控
**日期**: 2026-03-02
**脚本**: `stock_monitor.py`

---

## 问题 1：akshare 境外访问被封锁

**现象**
调用 `ak.stock_sh_a_spot_em()`、`ak.stock_sz_a_spot_em()`、`ak.stock_cy_a_spot_em()` 均抛出：
```
RemoteDisconnected / ConnectionError
```

**原因**
akshare 数据源（东方财富等）在中国大陆以外的网络环境下无法访问，请求被重置。

**解决方案**
完全弃用 akshare，改用 `yfinance` Screener：
```python
_CN_QUERY = yf.EquityQuery('or', [
    yf.EquityQuery('eq', ['exchange', 'SHH']),
    yf.EquityQuery('eq', ['exchange', 'SHZ']),
])
yf.screen(_CN_QUERY, sortField=..., sortAsc=..., size=50)
```
覆盖上证 + 深证约 7180 只股票，境外可正常访问。

---

## 问题 2：Yahoo Finance 股票名称为英文

**现象**
`yf.screen()` 返回的 `shortName` 字段均为英文，例如 `"Kweichow Moutai Co., Ltd."` 而非 `"贵州茅台"`。

**原因**
Yahoo Finance 数据库仅存储英文公司名称，不提供中文。

**解决方案**
引入腾讯财经 API（`qt.gtimg.cn`）作为中文名称来源：
- 境外可正常访问
- 返回格式：`v_sh600519="1~贵州茅台~600519~1455.02~..."`，字段 `[1]` 即中文名
- 实现懒加载 + 本地 JSON 缓存（`ashare_names.json`），30 天有效期
- 每次榜单刷新后仅补全缓存中缺失的代码，避免重复请求

---

## 问题 3：启动时全量缓存构建失败

**现象**
启动阶段通过 `yf.screen(size=2500)` 尝试一次性拉取所有 A 股代码以预填中文名缓存，静默失败，日志输出「无法获取股票代码列表」。

**原因**
`yf.screen()` 单次请求上限或网络超时，导致大批量请求失败。

**解决方案**
放弃全量预加载，改为懒加载策略：
```python
def fill_missing_names(codes: list) -> None:
    missing = [c for c in codes if c not in _NAMES]
    if not missing:
        return
    new_names = _fetch_names_tencent(missing)
    ...
```
每轮榜单数据获取后调用，仅查询当前榜单中尚未缓存的代码。

---

## 问题 4：每次刷新追加新表格，界面无法原地更新

**现象**
使用 `console.clear()` + `console.print()` 循环刷新，每次刷新会在终端追加一整张新表，滚动越来越长。

**解决方案**
改用 `rich.live.Live`（`screen=True` 模式），切换到备用屏幕缓冲区，实现原地全屏刷新：
```python
with Live(build_screen(...), console=console, screen=True, refresh_per_second=1) as live:
    while True:
        ...
        live.update(build_screen(...))
```

---

## 问题 5：高亮功能架构问题

**现象**
需要实现「涨跌幅变化 ≥ 0.25% 时触发 0.5 秒行高亮」，但原先数据获取与渲染耦合在一起，无法在不重新拉取数据的情况下二次渲染。

**解决方案**
将数据获取与渲染分离：
- `fetch_all_data()` 负责拉数据，返回纯数据结构
- `build_screen()` 负责纯渲染，接受 `highlighted: set` 参数
- `compute_highlighted()` 对比 `_prev_pct` 字典检测变化

刷新流程：
```
fetch_all_data() → compute_highlighted()
  → [有变化] live.update(高亮版) → sleep(0.5s)
  → live.update(正常版)
```

---

## 问题 6：Panel 底部内容被截断（只显示到第 47 行）

**现象**
榜单设置为 50 行，但终端中 Panel 只显示到第 47 行，底部 3 行被截断。

**原因**
榜单行数固定为 50，未考虑终端实际高度与固定 UI 元素（Panel 边框、自选股区块、Rule、表头、状态栏等）占用的行数。

**解决方案**
实现动态行数计算：
```python
def _leaderboard_size() -> int:
    OVERHEAD = 30  # Panel边框(2) + 自选股(18) + 空行(1) + Rule(1) + 表头区(4) + 状态栏(1) + 余量(3)
    return max(5, console.size.height - OVERHEAD)
```
`fetch_all_data()` 每次调用 `_leaderboard_size()` 动态传入 `size` 参数，随终端窗口大小自适应。

---

## 问题 7：启动时命令行与监控界面之间存在大段空白

**现象**
运行脚本后，上一条命令行提示符与监控界面之间有很大的空白距离，视觉上不连贯。

**原因**
原逻辑在进入 `Live` 之前先执行 `console.print("正在加载数据…")` 并完成全部数据拉取（耗时数秒），`Live(screen=True)` 启动时终端已滚动，形成空白间距。

**解决方案**
在 `Live` 启动时立即传入加载占位面板，将数据拉取移入 `with Live(...)` 块内部：
```python
with Live(_loading_panel(), console=console, screen=True, refresh_per_second=1) as live:
    cats_data, df_gain, df_loss, df_vol, all_pcts = fetch_all_data()
    ...
    live.update(build_screen(...))
```
`screen=True` 立即切换备用缓冲区，不再产生空白间距。

# Web UI 开发日志 v0.2
**日期**: 2026-03-02
**新增文件**: `stock_monitor_ui.py`、`templates/index.html`

---

## 架构概览

在终端版基础上新增独立 Web 服务，终端版保持不变。

| 项目 | 方案 |
|------|------|
| 后端 | Flask，直接 `import stock_monitor` 复用数据层 |
| 实时推送 | SSE（Server-Sent Events），多客户端订阅者队列 |
| 前端 | 原生 HTML + CSS + JS，无框架依赖 |
| 新闻抓取 | `feedparser` 解析 RSS/Atom |
| 启动 | `python stock_monitor_ui.py [--interval N] [--port N]` |

---

## 功能实现记录

### 1. Flask SSE 多客户端推送

每个 `/stream` 连接注册一个独立 `queue.Queue()`，后台线程通过 `_notify_subscribers(msg)` 广播事件（`refresh` / `news` / `live-news`），连接断开时自动从列表中移除，避免内存泄漏。

---

### 2. 行情数据

- 榜单固定 `LEADERBOARD_SIZE = 30`（终端版受窗口高度限制，Web 端固定）
- 新增国际期货面板：黄金（GC=F）、石油（CL=F）、白银（SI=F）、有色金属/铜（HG=F）
- 高亮行通过 CSS `@keyframes flash` 实现 0.6s 金色背景闪烁

---

### 3. 新闻瀑布流（上半区）

- 来源：BBC、NYTimes、Guardian、CNN、NBC（RSS/Atom）
- 通用抓取器 `_fetch_feeds(feeds)`，来源配置为 `list[dict]`，每条支持独立 `max`、`headers` 字段，新增来源只需追加一个 dict
- 去重：前端用 `Set` 记录已渲染的 `link`，仅插入新条目
- 新条目出现时触发 `@keyframes newsflash` 蓝色背景淡出动画

---

### 4. Live News 面板（下半区）

每个新闻源固定一行（共 5 行），显示该源最新头条。检测到 `link` 变化时更新内容并触发从上方滑入的翻页动画（`@keyframes liveFlip`，`translateY(-110%) → 0`，0.4s）。

**来源经历的调整：**

| 轮次 | 原因 | 结果 |
|------|------|------|
| 初始 | BBC/NYTimes/NBC/SkyNews/CCTV | BBC 无专属 Breaking RSS，内容重复 |
| 第一次替换 | BBC → AP（RSSHub） | AP 返回 403，RSSHub 被屏蔽 |
| 第二次替换 | Reuters + AP → CNBC + ABC | Reuters RSS 已关闭，AP 无效；CNBC/ABC 均可用 |

**最终 Live News 来源：** CNBC、ABC News、SkyNews、Al Jazeera、MarketWatch

---

### 5. 自动翻译

翻译优先级链：

```
DeepL Free API → Google Translate（免费接口）→ 原文（fallback）
```

- CJK 字符占比 > 30% 的标题直接跳过（如 CCTV 中文标题）
- 缓存：`trans_cache.json`，格式 `{"原文": {"cn": "中文", "ts": 时间戳}}`
- TTL：7 天，启动时自动清除过期条目，支持旧格式自动迁移
- 节流：每翻译一条新标题 sleep 50ms，避免触发频率限制
- 前端：显示 `title_cn`，原文放在 `<a title="">` tooltip

## 问题与修复

### 问题 1：`import time` 命名冲突

**现象**：`background_fetch` 中 `time.sleep()` 报错。
**原因**：`import time as time_module` 后遗漏了一处 `time.sleep()` 未同步改名。
**修复**：统一改为 `time_module.sleep(interval)`。

---

### 问题 2：DeepL 认证方式过期

**现象**：控制台持续打印 `[trans] DeepL error: 'translations'`。
**原因**：DeepL API v2 已废弃 `auth_key` 表单参数，改为要求 HTTP Header 认证。
**修复**：
```python
# 旧（已废弃）
data={"auth_key": DEEPL_API_KEY, "text": text, "target_lang": "ZH"}

# 新
headers={"Authorization": f"DeepL-Auth-Key {DEEPL_API_KEY}"},
data={"text": text, "target_lang": "ZH"}
```

---

### 问题 3：Live News 只显示 3 行而非 5 行

**现象**：面板只渲染出 3 个来源的行，另外 2 个空缺。
**原因 A**：Reuters（`feeds.reuters.com`）已永久关闭公开 RSS，AP（RSSHub）返回 403，两个来源抓取失败，对应行不被创建。
**原因 B**：剩余 3 行每行 `flex: 1` 撑满容器，超出可视区。
**修复**：替换失效来源（→ CNBC + ABC），并将每行改为 `flex: 0 0 20%` 固定占 20% 高度，确保 5 行始终可见。

---

### 问题 4：Live News 翻页动画未正确显示

**现象**：Live News 各行在内容更新时翻页动画（从上方滑入）未能正确触发或显示效果异常。
**状态**：待排查。可能原因为 CSS animation 重启时机问题（`classList.remove` + `offsetWidth` reflow + `classList.add` 的时序）或浏览器对 `overflow: hidden` 父容器的裁剪行为不一致。

---

### 问题 5：新闻板块自动刷新功能有待验证

**现象**：新闻瀑布流（120s 轮询）和 Live News（30s 轮询）的 SSE 推送与前端自动更新逻辑已实现，但实际运行中是否稳定触发、翻译缓存是否正确命中、断线重连后状态是否一致，均未经过完整的长时间验证。
**状态**：待测试。
