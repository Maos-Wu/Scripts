## 一、终端监控 v0.1（`stock_monitor.py`）

### 问题 1：akshare 境外访问被封锁

**现象**：调用 `ak.stock_sh_a_spot_em()` 等接口均抛出 `RemoteDisconnected / ConnectionError`。

**原因**：akshare 数据源（东方财富等）在中国大陆以外无法访问。

**解决**：完全弃用 akshare，改用 `yfinance` Screener：
```python
_CN_QUERY = yf.EquityQuery('or', [
    yf.EquityQuery('eq', ['exchange', 'SHH']),
    yf.EquityQuery('eq', ['exchange', 'SHZ']),
])
yf.screen(_CN_QUERY, sortField=..., sortAsc=..., size=50)
```
覆盖上证 + 深证约 7180 只股票，境外可正常访问。

---

### 问题 2：股票名称为英文

**原因**：Yahoo Finance 数据库仅存储英文公司名称。

**解决**：引入腾讯财经 API（`qt.gtimg.cn`）作为中文名来源，返回格式 `v_sh600519="1~贵州茅台~..."`。实现懒加载 + 本地 JSON 缓存（`ashare_names.json`，30 天有效期），每轮刷新后仅补全缓存中缺失的代码。

---

### 问题 3：启动时全量缓存构建失败

**原因**：`yf.screen(size=2500)` 单次请求量过大，静默失败。

**解决**：放弃全量预加载，改为懒加载：
```python
def fill_missing_names(codes: list) -> None:
    missing = [c for c in codes if c not in _NAMES]
    if not missing:
        new_names = _fetch_names_tencent(missing)
```

---

### 问题 4：每次刷新追加新表格

**解决**：改用 `rich.live.Live(screen=True)` 切换备用屏幕缓冲区，实现原地全屏刷新。

---

### 问题 5：高亮功能架构耦合

**解决**：将数据获取与渲染分离：
- `fetch_all_data()` 返回纯数据结构
- `build_screen()` 接受 `highlighted: set` 参数，纯渲染
- `compute_highlighted()` 对比 `_prev_pct` 字典检测变化

刷新流程：`fetch → highlight → live.update(高亮版) → sleep(0.5s) → live.update(正常版)`

---

### 问题 6：Panel 底部内容被截断

**原因**：榜单固定 50 行，未考虑终端实际高度与固定 UI 元素占用行数。

**解决**：实现动态行数计算：
```python
def _leaderboard_size() -> int:
    OVERHEAD = 30  # Panel边框 + 自选股 + Rule + 表头 + 状态栏 + 余量
    return max(5, console.size.height - OVERHEAD)
```

---

### 问题 7：启动时界面有大段空白

**原因**：`Live(screen=True)` 启动前执行了输出和数据拉取，终端已滚动。

**解决**：先启动 `Live`（立即切换备用缓冲区），再在 `with` 块内部拉取数据。

## 二、Web UI v0.2 初始实现（`stock_monitor_ui.py` + `index.html`）

### 架构

| 项目 | 方案 |
|------|------|
| 后端 | Flask，import stock_monitor 复用数据层 |
| 实时推送 | SSE（Server-Sent Events），多客户端订阅者队列 |
| 前端 | 原生 HTML + CSS + JS，无框架依赖 |
| 新闻抓取 | feedparser 解析 RSS/Atom |
| 启动 | `python stock_monitor_ui.py [--interval N] [--port N]` |

### Flask SSE 多客户端推送

每个 `/stream` 连接注册独立 `queue.Queue()`，后台线程通过 `_notify_subscribers(msg)` 广播事件（`refresh` / `news` / `live-news`），连接断开时自动移除，避免内存泄漏。

### 行情数据

- 榜单固定 `LEADERBOARD_SIZE = 30`
- 国际期货面板：黄金（GC=F）、石油（CL=F）、白银（SI=F）、铜（HG=F）
- 高亮行通过 CSS `@keyframes flash` 实现 0.6s 金色背景闪烁

### 新闻瀑布流（早期版本）

来源：BBC、NYTimes、Guardian、CNN、NBC（RSS/Atom），通用抓取器 `_fetch_feeds()`，前端 Set 去重，新条目触发蓝色背景淡出动画。

### Live News 面板（早期版本）

每个新闻源固定一行，显示最新头条，内容更新触发从上方滑入翻页动画。

**来源调整历史：**

| 轮次 | 变更 | 原因 |
|------|------|------|
| 初始 | BBC/NYTimes/NBC/SkyNews/CCTV | BBC 无 Breaking 专属 RSS，内容重复 |
| 第一次 | BBC → AP (RSSHub) | AP 返回 403，RSSHub 被屏蔽 |
| 第二次 | Reuters+AP → CNBC+ABC | Reuters RSS 已永久关闭，AP 无效 |

最终来源：CNBC、ABC、SkyNews、AlJazeera、MarketWatch

### 自动翻译

优先级链：DeepL Free API → Google Translate 免费接口 → 原文 fallback

- CJK 字符占比 > 30% 的标题直接跳过
- 缓存：`trans_cache.json`，7 天 TTL
- 节流：每条翻译后 sleep 50ms

### 问题与修复

| # | 现象 | 原因 | 修复 |
|---|------|------|------|
| 1 | `time.sleep()` 报错 | `import time as time_module` 后遗漏旧调用 | 统一改为 `time_module.sleep()` |

| 4 | Live News 翻页动画异常 | CSS animation 重启时机问题 | 待排查 |
| 5 | 新闻自动刷新稳定性 | 长时间运行未验证 | 待测试 |

## 三、Web UI 布局优化（Session 2）

### 环境配置

Python 3.14.3 环境下安装五个依赖（yfinance、rich、pandas、flask、feedparser），并将 Python Scripts 目录加入用户 PATH。

**yfinance 版本问题**：尝试降级至 0.2.51/0.2.48 均失败（`ValueError: Invalid field for Screener`，旧版不支持 `exchange/SHH`），锁定 `yfinance==1.2.0`。

### 响应式适配

- `body` 改为 `height: 100vh; display: flex; flex-direction: column; overflow: hidden`
- `#app` 增加 `flex: 1; min-height: 0`
- 所有嵌套 flex 容器加 `min-height: 0` / `min-width: 0` 防止溢出
- grid 列宽改为 `repeat(n, 1fr)`（自适应）

### 左右两栏布局

| 区域 | 值 |
|------|-----|
| 左栏 `#left-col` | `flex: 0 0 50%` |
| 右栏 `#news-panel` | `flex: 1; min-width: 0` |

### 左栏上下分割（35/65）

```html
<div id="main">

    <div class="top-row">

  </div>
```

### 表格行高自适应

```css
/* 上半：table 填满卡片，行高均分 */
.left-top .card table { flex: 1; height: 100%; }

/* 下半：榜单 lb-body 填满，table 行高均分 */
.left-bottom .lb-body { flex: 1; max-height: none; }
.left-bottom .lb-body table { height: 100%; }
```

### yfinance 限流问题

**现象**：左栏空白，后台静默挂起。
**诊断**：`YFRateLimitError: Too Many Requests`，冷却期约 15–60 分钟。
**处理**：等待解除，无需改代码。

### 关键 CSS 模式总结

| 场景 | 方案 |
|------|------|
| 全屏布局 | `body { height: 100vh; display: flex; overflow: hidden }` |
| 嵌套 flex 不溢出 | 每层加 `min-height: 0` |
| 表格行高自适应 | flex 父容器确定高度 + `table { height: 100% }` |
| 固定比例分割 | `flex: 0 0 X%` + `flex: 1` |
| 榜单不固定最大高度 | `flex: 1; max-height: none` |
