## 二、Bug 修复：A 股榜单未延伸至底部

### 根因

| 位置 | 代码 | 作用 |
|------|------|------|
| CSS | `#main { display: flex; flex-direction: column }` | flex 布局，子区按比例分配高度 |
| CSS | `#main { display: none }` | 初始隐藏 |
| JS | `style.display = 'block'` | **Bug**：内联样式优先级高于样式表，覆盖了 flex |

**结果**：`#main` 变为 block 容器，`.left-top { flex: 0 0 35% }` 和 `.left-bottom { flex: 1 }` 全部失效，底部大片空白。

### 修复

```javascript
// 修改前
document.getElementById('main').style.display = 'block';
// 修改后
document.getElementById('main').style.display = 'flex';
```

## 三、布局高度传递链全审查

```
100vh (body)
 └── #app (flex:1, flex-row)
      ├── #left-col (flex:0 0 50%, stretch高度)
      │    └── #main (width:100%, height:100%, flex-column)
      │         ├── .left-top  (flex:0 0 35%)
      │         │    └── .top-row (flex:1)
      │         │         ├── #watchlist-grid.grid-2
      │         │         └── #futures-panel
      │         └── .left-bottom (flex:1)
      │              ├── .section-rule
      │              ├── #leaderboard-grid (flex:1, grid-template-rows:1fr)
      │              │    └── .card × 3 → .lb-body (flex:1) → table (height:100%)
      │              └── .status-bar
      └── #news-panel (flex:1, flex-column)
           ├── .news-half（新闻头条, flex:1）
           └── .news-half（Live News, flex:1）
                └── #live-list (flex-column) → .live-row (flex:0 0 20%) × 5
```

**关键规律**：
- 每层需父层提供"确定高度"才能使用 `flex:1` 或 `height:100%`
- `min-height: 0` 阻止 flex 默认的 `min-height: auto` 撑破边界
- CSS 样式表 < JS 内联样式，`display` 类型必须与内联样式保持一致

`.status-bar` 贴底依赖 `#leaderboard-grid { flex:1 }` 将其挤到底部，而非绝对定位。

## 四、PyInstaller 打包

### 打包方案

`--onedir` 模式，生成 `dist\EndfieldPanel\` 目录，对方无需安装 Python，双击 `启动.bat` 即可运行。

### 问题 1：exe 启动无反应

**原因**：使用了 `--noconsole`，隐藏了所有输出，用户不知道需手动访问 `http://localhost:5000`（服务实际已在运行）。

**解决**：去掉 `--noconsole`；打包后自动生成 `启动.bat`，等待 3 秒后自动打开浏览器。

### 问题 2：股票数据全部显示 N/A

**根本原因**：`curl_cffi` 无法加载 SSL 证书：
```
curl_cffi.requests.exceptions.SSLError: curl: (77) error setting certificate verify locations:
  CAfile: F:\GitHub project\Scripts\dist\帝江号终端\_internal\certifi\cacert.pem
```

libcurl 的 C 库不支持路径中的中文字符（`帝江号终端`），证书加载失败，所有 HTTPS 请求均失败。

**解决**：
1. 输出目录改为纯 ASCII 名称 `EndfieldPanel`
2. 启动时显式设置 SSL 证书环境变量：

```python
if _IS_BUNDLE:
    import certifi as _certifi
    _ca = _certifi.where()
    os.environ['CURL_CA_BUNDLE']     = _ca
    os.environ['SSL_CERT_FILE']      = _ca
    os.environ['REQUESTS_CA_BUNDLE'] = _ca
```

**教训**：libcurl 的 `CURLOPT_CAINFO` 路径在 Windows 下不支持非 ASCII 字符，exe 名称和输出目录必须纯 ASCII。

### 问题 3：旧 exe 未关闭导致打包失败

**原因**：`rmdir /s /q dist` 时旧进程仍占用 `.dll`/`.pyd` 文件。

**解决**：重新打包前先 `taskkill /f /im EndfieldPanel.exe`。

### 最终打包命令

```bat
pyinstaller ^

  --hidden-import=pkg_resources.py2_warn ^
  --hidden-import=multitasking ^
  --hidden-import=peewee ^
  --hidden-import=frozendict ^
  --hidden-import=appdirs ^
  --hidden-import=curl_cffi ^
  --hidden-import=curl_cffi.requests ^
  --collect-all yfinance ^

  --copy-metadata yfinance ^

```

### 后续修复（2026-03-04）

**Bug 1：点击链接后 New! 不消失**

**根因**：`_dismissedNew` 仅在点击 badge 本身时写入，`<a>` 上无监听器，跳转后 badge 在下次 `renderNews` 重建。

**修复**：提取公共函数 `dismiss()`，同时注册到 badge 和 `<a>`：

```javascript
const dismiss = () => { _dismissedNew.add(item.link); badge.remove(); };
badge.addEventListener('click', (e) => { e.preventDefault(); e.stopPropagation(); dismiss(); });
a.addEventListener('click', dismiss, { once: true });
```

**Bug 2：显示窗口过短**

300 秒（5 分钟）改为 900 秒（15 分钟）：

```javascript
const isNew = item._ts > 0 && (now - item._ts) <= 900 && !_dismissedNew.has(item.link);
const remaining = Math.max(100, (900 - (now - item._ts)) * 1000);
```

**新增：Live News 块也支持 New! 角标**

在 `_updateLiveInline()` 中复用相同逻辑（共享 `_dismissedNew`）：每次切换条目时先移除旧 badge，再对当前条目判断是否需要显示。

## 十、RSS 并发抓取重构

**问题**：串行请求 8 个来源最坏情况 96 秒，超过抓取间隔。

**方案**：`ThreadPoolExecutor(max_workers=len(feeds))` 并发，实际耗时降至约 12 秒。

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

def _fetch_feeds(feeds: list) -> list:
    with ThreadPoolExecutor(max_workers=len(feeds)) as executor:
        futures = [executor.submit(_fetch_one_feed, cfg) for cfg in feeds]
        for future in as_completed(futures):
            items.extend(future.result())
    items.sort(key=lambda x: x.get("_ts", 0), reverse=True)
```

## 十四、扩展面板：哔哩哔哩视频区（规划，2026-03-04）

### 功能目标

|---------|----------------|---------|
| 财经区 | B站财经分区 | 158 |

| 科技区 | B站科学科普 | 122 |

| 米哈游区 | 搜索 tag：明日方舟 + 终末地 | — |

> **澄清记录**：规划时询问"终末地"指哪款游戏。
> 确认：指《**明日方舟：终末地**》（Arknights: Endfield），明日方舟系列续作。
> 米哈游区实际为「鹰角网络区」，搜索关键词：`明日方舟`、`终末地`，合并结果按发布时间排序。

### 实现方案
- **CORS 问题**：B站 API 不允许浏览器跨域直调，**必须走后端代理**。
- **后端**：新增 Flask 路由 `/api/bilibili?rid=xxx`（或 `?type=mihayo`），Python 向 B站 API 发起请求，5 分钟缓存，返回精简 JSON。
- **普通分区 API**：`api.bilibili.com/x/web-interface/newlist?rid={rid}&type=1&pn=1&ps=20`
- **搜索 API**（米哈游区）：`api.bilibili.com/x/web-interface/search/type?search_type=video&keyword={kw}&order=pubdate`
- **WBI 风控**：若接口返回 `-352` 风控错误，需在后端实现 WBI 签名（MD5 混淆，算法已公开）。
- **前端布局**：Tab 栏选分区 → 视频列表（标题 + UP主 + 播放量） → 点击展开 iframe 内嵌播放器。

### 负载评估
| 项目 | 数值 |
|------|------|
| 分区总数（含米哈游 2 次搜索） | 6 次 B站 API 调用 / 缓存周期 |
| 缓存时长 | 5 分钟 |
| 单次请求耗时（B站侧） | 约 100–500 ms |
| 对本机后端 CPU | 极低（仅 HTTP 转发 + JSON 裁剪） |
| 对本机带宽 | 极低（列表数据 < 50 KB/次） |
| iframe 播放流量 | 直连 B站 CDN，**不经过本机后端** |

结论：**对本机后端几乎无额外负载**，瓶颈在 B站服务器的速率限制（实测日常请求量远低于限额）。
